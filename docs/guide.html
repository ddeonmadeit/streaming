Before you start

Set aside ~45 minutes for the one-time build. After that, going live is just “open Moblin → tap start.” You’ll do the server steps on a computer; the Moblin steps on your iPhone. Keep the irl-stream package I sent you handy.

00

What you’ll need

  • An iPhone with the Moblin app (free on the App Store).
  • A computer to set up the server (just for setup — not needed while streaming).
  • A free Oracle Cloud account (a card is used for ID check; the Always-Free tier isn’t charged).
  • Accounts on Twitch, Kick, and TikTok.
  • The irl-stream package (the .tar.gz I sent).
The one real catch — TikTok

TikTok LIVE needs 1,000+ followers and separate third-party (RTMP) access. If your account doesn’t have it yet, Twitch and Kick still work perfectly — TikTok just won’t connect until you qualify. Nothing in this guide (or any paid tool) can bypass that.

01

Create the free cloud server

This is the machine that duplicates your stream to all three platforms — the piece paid services charge for. You’ll run it yourself for free.

  1. Go to cloud.oracle.com and sign up. Choose your home region and complete the card verification. Stay on the free account — don’t upgrade to Pay-As-You-Go.
  2. In the console: Compute → Instances → Create instance.
  3. Under Shape, click Change shape → Ampere (ARM) → VM.Standard.A1.Flex. Set 4 OCPUs and 24 GB RAM (inside the Always-Free allowance).
  4. Set Image to Ubuntu 22.04 (or newer).
  5. Make sure a public IPv4 is assigned, then download the SSH key when prompted.
  6. Click Create. When it’s running, copy the public IP address — you’ll use it as SERVER_IP everywhere below.
02

Open the streaming port

Your phone sends video in over UDP port 8890. Oracle blocks it in two places, so you open it in both. (Only this one port is exposed — everything else stays private.)

  1. Cloud firewall: in the console go to Networking → Virtual Cloud Networks → your VCN → its Subnet → Security List → Add Ingress Rule, and add: 
    Source CIDR:  0.0.0.0/0
Protocol:     UDP
Dest. port:   8890
  2. Server firewall: connect to the VM from your computer’s terminal (replace your-key and SERVER_IP): 
    ssh -i your-key.key ubuntu@SERVER_IP
    then run: 
    sudo iptables -I INPUT 6 -p udp --dport 8890 -j ACCEPT
sudo netfilter-persistent save
Windows note

No ssh? Use the free PuTTY app, or Oracle’s built-in Cloud Shell (the >_ icon in the console).

03

Install Docker & upload the package

Docker runs the whole setup with one command and restarts it automatically if anything hiccups.

  1. On the server (still SSH’d in), install Docker: 
    sudo apt-get update
sudo apt-get install -y docker.io docker-compose-plugin
sudo usermod -aG docker $USER
    Then type exit, reconnect with ssh again so the permission takes effect.
  2. From your computer (in the folder holding docker-compose.yml), upload the package: 
    scp -i your-key.key -r . ubuntu@SERVER_IP:/home/ubuntu/irl-stream
  3. Back on the server, enter the folder: 
    cd irl-stream
04

Grab your three stream keys

Each platform gives you a server URL and a secret stream key. You’ll join them together in the next phase. Treat keys like passwords — don’t share them.

PlatformWhere to find it
Twitchdashboard.twitch.tv → Settings → Stream → Primary Stream key. Server is fixed: rtmp://live.twitch.tv/app/
KickKick creator dashboard → Stream → copy the Stream URL and Stream Key (Kick uses rtmps://…)
TikTokTikTok LIVE (needs LIVE + RTMP access) → Go LIVE via third-party → copy the Server and Key
05

Add your keys & a “BRB” clip

  1. Copy the settings template and open it: 
    cp .env.example .env
nano .env
  2. Fill in each line. PUBLISH_PASS is a secret you invent (no spaces or colons) — your phone uses it to connect. For each platform, paste the server URL with the key added on the end, e.g.: 
    PUBLISH_PASS=my-long-secret-2718
TWITCH_URL=rtmp://live.twitch.tv/app/YOUR_TWITCH_KEY
KICK_URL=rtmps://host/app/YOUR_KICK_KEY
TIKTOK_URL=rtmp://host/live/YOUR_TIKTOK_KEY
    Save in nano with Ctrl-O, Enter, then Ctrl-X. Leave a platform blank to skip it.
  3. Add a short “be right back” video at media/brb.mp4. No clip? Make a 10-second card right on the server (needs a background bg.jpg, or swap in a black screen): 
    ffmpeg -loop 1 -i bg.jpg -f lavfi -i anullsrc=r=48000:cl=stereo -t 10 \
  -vf "scale=1280:720,fps=60,drawtext=text='Reconnecting…':\
fontcolor=white:fontsize=64:x=(w-text_w)/2:y=(h-text_h)/2" \
  -c:v libx264 -pix_fmt yuv420p -c:a aac -shortest media/brb.mp4
Want it done for you?

I can render a ready-made “Reconnecting…” BRB video (720p60, silent audio track) and send it — just ask and drop it into media/.

06

Start the server

One command builds and launches everything. It stays running 24/7 and restarts itself on reboot.

docker compose up -d --build
docker compose logs -f

Within a minute you should see your BRB clip start looping to your platforms — meaning the pipe to Twitch/Kick/TikTok is open and holding. (Press Ctrl-C to stop watching the logs; the server keeps running.)

07

Set up Moblin on your iPhone

Now the phone side. In Moblin, add a new stream and enter exactly these:

ProtocolSRT (or SRTLA if you bond connections)
URLsrt://SERVER_IP:8890
Stream IDpublish:live:publisher:YOUR_PUBLISH_PASS
Latency2000 ms  (3000–5000 on shaky cellular)
Resolution / FPS1280×720 · 60 fps
Bitrate~6000 kbps  (turn on adaptive bitrate)

Swap SERVER_IP for your server’s IP and YOUR_PUBLISH_PASS for the secret you set in .env. That’s the whole phone setup.

08

Go live & test the safety net

  1. Tap Go Live in Moblin. Within ~1–2 seconds the server switches from the BRB card to your camera — you’re now live on all three platforms at once. 🎉
  2. Test the failover: put your phone in airplane mode for a few seconds. Your platforms should keep showing the BRB loop instead of ending the stream. Turn signal back on — it flips back to your camera automatically. That’s the “never get dropped” feature working.
Why this works

The server keeps one unbroken connection to each platform the whole time and just swaps what it sends — your camera when you’re connected, the BRB loop when you’re not. The platforms never see the feed stop, so they never end your broadcast.

09

See all your chats — on both phones

Two free tools; use them together so you and your friend both watch one merged chat.

  1. On your (streamer) phone — Moblin’s built-in chat. In Moblin’s chat settings, add each platform (Twitch, Kick, YouTube…). It overlays them as one combined list while you stream.
  2. On both phones — Social Stream Ninja (socialstream.ninja), free and browser-based:
    • Open the site in any browser and create a session.
    • Connect each platform’s chat (Twitch, Kick, TikTok, YouTube all supported).
    • Open the dashboard view — it merges every message into one feed, labeled by platform.
    • Send that dashboard link to your friend — you both open it and see the identical combined chat live. No install, no watermark.
Handy

Add the Social Stream dashboard link to your phone’s home screen so it opens like an app.

10

Everyday commands

Run these on the server whenever you need them:

# after changing keys (.env) or the BRB clip
docker compose restart restreamer

# stop / start the whole thing
docker compose down
docker compose up -d

# watch it switch between live and BRB in real time
docker compose logs -f restreamer

You’re fully live when…

  • The server logs show the BRB loop playing before you even connect the phone.
  • Tapping Go Live in Moblin switches all platforms to your camera within ~2 s.
  • Airplane-mode test keeps the platforms alive on the BRB card instead of ending.
  • Twitch and Kick both show your stream (TikTok too, if your account qualifies).
  • Your combined chat is open on your phone and your friend’s.
